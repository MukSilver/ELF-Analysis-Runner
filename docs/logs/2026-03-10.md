- commit은 로컬 작업 기록을 남기는 것이다.
- push는 로컬 commit을 GitHub에 업로드하는 것이다.
- README.md는 프로젝트를 설명하는 문서이다.
- VS Code에서도 Git 작업을 할 수 있다.

## 막힌 점
- GitHub/레포 개념 자체가 익숙하지 않았음
- VS Code에서 GitHub 로그인 위치를 바로 찾기 어려웠음
- push할 때 git remote에 인증하지 못했습니다 에러가 났음
- VS Code UI에서 커밋/동기화 흐름이 처음엔 헷갈렸음

## 해결한 점
- 레포 역할, commit / push 개념을 직접 해보면서 이해함
- VS Code 로그인에 집착하지 않고 터미널 git push -u origin main으로 우회해서 해결함
- README 수정 → commit → push → GitHub 반영까지 한 사이클을 끝냄
- 로컬 작업 환경과 GitHub 원격 저장소를 실제로 연결함
- 하루 기록을 남길 템플릿까지 만들어서 앞으로 안 꼬이게 구조를 잡음
